## はじめに

私はスクリーンショットや紙の研修資料のPDFファイルから文字起こしをする際に、**Google Cloud Vision API** を用いた自作ツール **VisionOCR** を使っています。

**Gemini** でPDFファイルのOCRを試みましたが思ったよりもうまくいかず、結果として **Google Cloud Vision API** を選びました。

この記事では、LLMでOCRを試したけれど思ったような精度が出ずに悩んでいる方、あるいはこれからOCRツールを選ぼうとしている方に向けて、気づきを共有しようと思います。

https://github.com/yokamoto5742/VisionOCR

## 結論

長文のOCRには **Google Cloud Vision API** が向いています。LLMによるOCRは、1つの画像ファイルや表・レイアウト保持には強いものの、複数ページのPDFファイルでは **「文章がまるごと抜け落ちる」** という現象が起きました。

LLMは何でも使えて万能に見えますが、用途によっては従来のGoogle Cloud Vision APIのほうが正確にOCR処理ができます。

## GeminiのOCRで文章が消える現象

衝撃だったのは、ある時10ページ程度のPDFを Gemini で文字起こししたら、**中間の数ページの内容がそのまま抜け落ちた** ことです。

- 「文字が別の単語に置き換わる」というよりは **「文章ががっさり抜け落ちる」**
- 抜けた箇所に欠落を示すマーカーや断りは特になく、**出力結果だけ見ると"全ページのOCRが成功している"ように見える**
- 同じPDFを Cloud Vision API に通すと、全ページ取得できた

「もっともらしい別の言葉に置き換わる」よりも、**OCRできた部分は正しいが一部欠落しているある方がわかりにくい**と感じました。プロンプトで｢全文正確に出力してください｣と指示しても、残念ながら似たような結果に終わりました。

## なぜこうなるのか

- **Cloud Vision API**: 画画像から文字を機械的に検出・抽出する**「転写」エンジン**。意味の解釈はせず、認識したものをそのまま出力するため、文脈による文字の省略は起きません。ただし、手書きや低解像度などで認識困難な文字は欠落することがあります。

- **Gemini**: 文脈を踏まえてテキストを再構成する**「生成」エンジン**。入力を逐語的に書き写すのではなく「確率的に最もらしい応答」を作るため、長文になるほど「全文転写」ではなく「要約・省略」の挙動が働きやすくなります。これは他のマルチモーダルLLMでも同じことが言えます。

Cloud Vision APIは検出した文字をそのまま返しますが、Gemini は確率的なテキスト生成を行うため、入力が長くなるほど「全文を逐語的に出力する」よりも「要点をまとめて返す」挙動が出やすくなります。そのため、完全な転写はOCR専用APIの方が適していると思われます。

## 比較表

| 観点 | Cloud Vision API | Gemini |
| --- |------------------| --- |
| 長文の網羅性 | ◎ 基本は抜けない | △ 抜けることがある |
| 手書き・半角カナ | ◎                | ○ |
| 表・レイアウト保持 | △ バラバラの行になりがち    | ◎ Markdown化が得意 |
| 翻訳・整形の同時実行 | ×                | ◎ プロンプトで指示可能 |
| ハルシネーション | なし               | あり |
| レスポンスの安定性 | ◎ 構造が固定          | △ プロンプト調整が必要 |

## OCRでどちらを選ぶか

:::message
**OCR結果の欠落や改変がある程度許容されるかかどうか** が選択基準の1つと思います。
:::

- 正確さが要求される業務文書や研修資料 → **Cloud Vision API**
- 表をMarkdown化したい / 翻訳も同時にしたい / 多少の改変は許容できる → **Gemini** などのマルチモーダルLLM

私は前者の用途が多いので、基本的には Cloud Vision API を利用しています。

## VisionOCR について

上記を踏まえて開発したデスクトップアプリケーションが **VisionOCR** です。

### 主な機能

- 画面の任意の領域をマウスドラッグで選択してOCR実行
- PDFファイルのテキスト抽出（複数ページ対応）
- 日本語の手書き文字・半角カタカナに対応
- テキスト編集機能(読点・句点・改行・空白の除去)
- 追記/上書きモードの切り替え
- OCR結果のクリップボードコピー・ファイル保存

### 動作環境

- Windows
- Python 3.13以上
- Google Cloud Vision API のサービスアカウント認証情報

詳細は VisionOCR の README を参照してください。

https://github.com/yokamoto5742/VisionOCR

## おわりに

LLMにはOCR処理に加えて出力結果のMarkdown化や翻訳など、OCR専用APIにはできないことがあります。一方で、**長文を正確に書き起こす**というシンプルな作業では、OCR専用APIに軍配が上がることもあります。

「LLMは既存の技術をすべて置き換える」と思われがちですが、現時点では「確実な転写は Cloud Vision API」「構造化や同時翻訳は Gemini」と、用途に合わせて使い分けるのがよいと思います。

LLMのOCR処理で出力がうまくいかずに困っている方は、一度 Cloud Vision API を試すのがよいかもしれません。VisionOCR がその一助になれば幸いです。
